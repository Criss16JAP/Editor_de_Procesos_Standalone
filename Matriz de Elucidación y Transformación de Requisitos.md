# MATRIZ DE ELUCIDACIÓN Y TRANSFORMACIÓN DE REQUISITOS (OUTCOME A)
 
Este documento demuestra el cumplimiento del primer resultado del proceso de la ISO 12207, el cual exige que las necesidades, expectativas y escenarios operacionales de los usuarios finales se transformen formalmente en un conjunto definido de requisitos técnicos del sistema.
 
| ID Necesidad | Necesidad del Cliente / Situación de Origen | Capacidad Técnica e Interfaz Solicitada | Requisitos del Sistema Asociados (SyRS) |
| :--- | :--- | :--- | :--- |
| **NN-01** | **Eliminación de plantillas manuales:** Las organizaciones documentan procesos en texto libre, hojas de cálculo o diapositivas, dificultando mantener una estructura uniforme y provocando que falten datos. | Interfaz de usuario interactiva especializada que automatice la captura de campos tipificados mediante un entorno estructurado guiado. | **RF-01** a **RF-05** (Operaciones básicas)<br>**RF-03** (Elementos ISO 24774)<br>**RNF-02**, **RNF-03** (Usabilidad) |
| **NN-02** | **Garantía de completitud:** Las descripciones manuales suelen dejar elementos incompletos o actividades y tareas desconectadas del propósito real del proceso. | Motor de validación semántica integrado localmente que analice el estado de los campos obligatorios e intercepte las funciones de persistencia si hay omisiones. | **RF-09** (Validación mandatoria)<br>**RF-10** (Detección estructural)<br>**RF-11** (Bloqueo persistencia)<br>**RNF-03** (Mensajes descriptivos) |
| **NN-03** | **Conservación de relaciones jerárquicas:** Dificultad para mantener enlaces lógicos de composición o secuencia entre elementos. | Modelo de datos gestionado mediante estructuras jerárquicas en memoria y reordenamiento dinámico directo en la interfaz visual. | **RF-04** (Mapeo jerárquico)<br>**RF-05** (Reorganizar secuencia)<br>**RNF-04** (Interacción de ordenamiento) |
| **NN-04** | **Reutilización e intercambio distribuido:** Los archivos tradicionales dificultan reutilizar procesos en otros contextos o transferirlos entre miembros del equipo. | Sistema de persistencia basado exclusivamente en la generación de archivos universales e intercambiables (`.pro`) estructurados internamente. | **RF-06**, **RF-07** (Exportar/Importar `.pro`)<br>**RF-08** (Continuidad)<br>**RNF-01** (Abstracción de datos) |
| **NN-05** | **Salidas consistentes para distribución:** Problemas de formato y desconfiguración visual al imprimir descripciones de procesos para revisión o archivo. | Componente de renderizado de solo lectura desacoplado de la edición, con soporte directo de parametrización para impresión e integración a PDF. | **RF-12** a **RF-14** (Vista previa y orientación)<br>**RF-15** (Exportación a PDF)<br>**RNF-07** (Eficiencia de renderizado) |
